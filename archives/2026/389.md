## 前言

在写代码这件事上，我是有点强迫症的，代码格式便是其中之一。

### 工具

个人常用组合是：

- **ESLint** - 代码质量检查
- **Prettier** - 代码格式检查
- **Stylelint** - CSS 质量检查（含格式化）

相较之下，比较纯粹的是 Prettier，专注于格式化。而 ESLint、Stylelint 由于历史原因，它除了做质量检查，其实也做格式化的事。所幸的是，它们也逐渐“去掉”格式化相关的规则，专注于质量检查。

- 在 ESLint 9 后，移除了所有格式化相关的规则。
- 在 Stylelint 15 后，移除了 76 条格式化相关的规则。 

其后由社区接盘，通过 Plugin 的方式继续提供：

- [ESLint Stylistic](https://eslint.style/guide/getting-started)
- [Stylelint Stylistic](https://github.com/stylelint-stylistic/stylelint-stylistic)

Prettier 是“固执己见”的，足够简单。相反，有定制化需求就比较困难，需通过 Plugin 自行实现。当然，它也不是完美的，如托尼老师 antfu 文章《[为什么我不使用 Prettier](https://antfu.me/posts/why-not-prettier-zh)》指出的一些问题，确确实实有困扰过，但接受它的不完美就还好吧。突然想起，当初刚开始用的时候也有类似的[问题](https://segmentfault.com/q/1010000023464230)。

这个过程，其实用起来挺不爽的，要配置多个工具，还要解决它们之间的冲突问题。

> 唤醒一下沉睡的记忆：此前解决 ESLint 与 Prettier 冲突的问题，有两种方法：一是配置 [eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) 屏蔽相关冲突规则，将格式化交由 Prettier 处理。二是使用 [eslint-plugin-prettier](https://github.com/prettier/eslint-plugin-prettier) 将不符合 Prettier 格式化规则的报告成 ESLint 错误，利用 ESLint AutoFix 自动修复。

后来又出现了 [Biome](https://biomejs.dev/guides/getting-started/)、[Oxlint](https://oxc.rs/docs/guide/usage/linter.html) 这种集 linter 和 formatter 于一体的工具，好处是不用配置那么多工具了。缺点是它们目前生态没那么繁荣。

### 小程序生态

前面提到这些，在标准 Web 里都有完善的插件提供支持。但小程序生态，它带一些特有的规则，不完全兼容。

罄竹难书：

- WXML 支持插值语法。类似于 Vue，但又不完全相同，如属性插值。
- WXML 支持嵌入 `<wxs>`，类似于 HTML 的 `<script>` 元素，但现有的 Parser 认不出来，原因是它不是标准 Web 元素。
    - 在 Prettier 用 `parser: 'html'` 或 `parser: 'vue'` 遇 `<wxs>` 会报错，前者还会将 `<wxs>` 当作文本内容产生不合法的换行，导致内置 JS 语句语法错误，编译失败。
    - 在 Stylelint 用 `customSyntax: 'postcss-html'` 遇 `<wxs>` 会报错。
- WXML 的元素属性不允许换行，否则会编译失败，而 HTML 是兼容的。
    - 对于超过 Prettier 指定行宽的属性值会产生换行。
    - 对于文本节点插值，由于 HTML 不支持插值语法，Prettier 可能会在非预期地方进行换行导致插值表达式语法错误
- WXML 有一套特有的标签、单位，而不是 HTML 中的 `<div>`、`<span>` 等，因此 Stylelint 默认认不出这些特有特性：
    - 遇到 `page {}` 会抛出 `selector-type-no-unknown` 错误
    - 遇到 `rpx` 会抛出 `declaration-property-value-no-unknown` 错误
- WXML 的属性支持 `style="{{style}}"`、`style="font-size: {{fontSize}}rpx"`、`style="font-size: 10rpx; {{color}}; font-Weight: 600"` 等特有的语法，对于 postcss-html 解析内联样式有点蛋疼。
- ...

所以，无论按 HTML，还是 Vue 去解析 WXML 都不太行，怎么破？

## 解决方案

唯有自实现一个 Parser 去解析，凑巧不是有 AI 嘛，于是...

- [@tofrankie/prettier-plugin-wxml](https://github.com/tofrankie/prettier-plugin-wxml) - 用于 Prettier
- [@tofrankie/postcss-wxml](https://github.com/tofrankie/postcss-wxml) - 用于 Stylelint

大致思路：

1. 将 WXML 拆开为 wxs 和 wxml（指无 wxs 的 wxml）
2. 把 wxs 按 JS 形式进行格式化
3. 将 wxml 按 Vue 形式进行格式化
4. 用 angular-html-parser 对格式化后的 wxml 提取插值
5. 按 JS 形式对插值表达式进行格式化，完成后替换回去
6. 用 angular-html-parser 解析所有属性，遍历将多行折叠为单行
7. 将格式化后的 wxs 替换回去
