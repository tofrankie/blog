## 背景

今天使用 Claude Sonnet 4.5 模型的时候，提示：

```
Model not available
This model provider doesn't serve your region.Visit https://docs.cursor.com/account/regions for more information
```

根据 Cursor 官网说明：**部分模型提供商有基于位置的限制，这意味着某些模型可能在你所在地区不可用。** [更多](https://cursor.com/cn/docs/account/regions)

> 兜底选择：可以继续使用 [Auto](https://cursor.com/cn/docs/models#auto) 模式，Cursor 会为每次请求自动选择可用模型。

## 解决方法一

> 参考 [cursor #3372](https://github.com/cursor/cursor/issues/3372)

1. 梯子开启 TUN 模式（[参考](https://github.com/tofrankie/blog/issues/382)）
2. 在 Cursor Settings - Network 处，将 HTTP Compatibility Mode 选项改为 HTTP/1.1 或 HTTP/1.0
3. 重启 Cursor

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1768208408853.png)

## 解决方法二（推荐）

使用方案一，发现有两个比较蛋疼的问题：

- 使用 Clash Verge 太占 CPU 了，务必关闭「浏量图显」
- 经常会 Taking longer than expected...，可能是使用 HTTP 1.x 的原因吧

参考《[Cursor 地区限制完全解决指南：This Model Provider Doesn't Serve Your Region 修复方案](https://www.cursor-ide.com/blog/cursor-region-restriction-vpn-guide)》代理配置方式。

1. 将上述 Network 改回默认的 HTTP/2
2. 在你的 `settings.json` 添加以下配置

```json
{
  "http.proxy": "socks5://127.0.0.1:7897", // Clash 默认端口是 7890, Clash Verge 的默认端口是 7897
  "http.proxyStrictSSL": false,
  "http.proxySupport": "on",
}
```

> 实测还是要借助 Clash Verge 的 TUN 模式。
