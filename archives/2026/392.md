## 前言

以前，我们快速了解新项目的方式是：项目文档、使用文档、源码。现在把文档和源码丢给 AI，简单写个 Prompt 就能快速吐出答案。如果要生成包含架构图、代码摘要等结构化的 Wiki 文档，肯定要好好打磨 Prompt。

目前，以下这些产品提供了 Wiki 文档生成的能力，包括架构图、源文件链接、代码摘要等。

- [DeepWiki](https://deepwiki.com/) - Devin
- [Code Wiki](https://codewiki.google/) - Google
- [Zread](https://zread.ai/) - 智谱

优点：
- 免费
- 对公开仓库友好

缺点：
- 非实时
- 对私有仓库不友好，尽管
- 自动生成不支持人工干预，甚至无法修改
- 前两者生成的英文文档对国内用户不友好，但可使用中文问答

> Code Wiki 前不久好像推出了实时代码的能力 [详见](https://codewiki.google/faq)

> DeepWiki 支持一定程度地干预 Wiki 文档生成，但需要有维护者权限，需要将 `.devin/wiki.json` 添加到项目中 [详见](https://docs.devin.ai/work-with-devin/deepwiki#steering-deepwiki)

> 对于私有仓库，DeepWiki 支持在 [Devin AI](https://app.devin.ai/) 对话里生成（使用 GitHub 账号登录）。Code Wiki 则以 Gemini Extension 的形式提供在本地生成（加入 [Waitlist](https://developers.google.com/profile/badges/community/sdlcagents/gca-agents)）。Zread 提供了手动 GitHub 授权方式。

## 本地项目

对于暂未公开或无法公开的项目，我们可以借助一些工具在本地生成结构化的 Wiki 文档。

本文介绍的是 [Zread CLI](https://zread.ai/cli)，官方文档已经写得很清楚了，不再重复。

1. 安装

```bash
$ pnpm add zread_cli -g
```

2. 配置

```bash
$ zread config # 交互式
```

也可以直接修改文件 `~/.zread/config.yaml`（比如），没有则新增。

```json
language: zh
doc_language: zh
llm:
  provider: custom
  model: gpt-5.4-mini
  api_key: sk-fake
  base_url: https://example.ai/v1
concurrency:
  max_concurrent: 1
  max_retries: 0
```

对于官方订阅用户来说，里面内置了部分提供商，也可以使用 API Key 进行登录。

> 如果使用中转站的朋友，且频繁切换上游，建议将 CC Switch 上的供应商统一使用 custom，这样才能确保切换不同的 API Key 时，所有对话不“丢失”。

3. 生成文档

执行 `zread generate` 生成文档，耐心等待所有完成。

4. 浏览文档

使用 `zread browse`，将 current 所对应的文档版本用浏览量打开，方便阅读查看。

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/5/1778555361829.png)

5. 更新文档

后续迭代，还能继续使用 `zread generate` 最新版本的文档。
