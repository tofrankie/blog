### 调试扩展时关闭其他扩展

目的是在扩展开发宿主窗口关闭所有扩展以减少干扰，便于排查问题。在 `.vscode/lanuch.json` 的 `args` 中加入 `--disable-extensions`，示例：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug Extension",
      "type": "extensionHost",
      "request": "launch",
      "args": ["--disable-extensions", "--extensionDevelopmentPath=${workspaceFolder}"],
      "outFiles": ["${workspaceFolder}/dist/extension/*.js"],
      "preLaunchTask": "npm: dev"
    }
  ]
}
```

### 无法打开扩展开发宿主窗口

如果在调试面板启动调试，久久不能打开扩展开发宿主窗口，提示“正在生成...”，并且任务执行并无报错，通常是未正确配置 `problemMatcher`。

任务运行的结果是由属性 `problemMatcher` 来控制的。可以选择 VS Code 内置的，或者其他插件提供的结果分析器，甚至可以自己书写结果分析器来分析任务运行结果。

比如基于 tsdown 开发，默认情况下无法判断运行结果，可以根据 tsdown 的运行结果自行编写。

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "dev",
      "type": "shell",
      "command": "npm",
      "args": ["run", "dev", "--loglevel", "silent"],
      "isBackground": true,
      "problemMatcher": {
        "base": "$tsc-watch",
        "background": {
          "activeOnStart": true,
          "beginsPattern": "^.*$",
          "endsPattern": "([Bb]uilt|[Ss]uccess|[Ff]inished|[Dd]one|[Ww]atching)"
        }
      },
      "group": {
        "kind": "build",
        "isDefault": false
      },
      "presentation": {
        "reveal": "silent",
        "panel": "shared"
      }
    }
  ]
}
```

### 本地资源加载问题

Webview 在隔离的上下文中运行，无法直接访问本地资源，这也是出于安全原因考虑。

默认是 `default-src 'none'`，无法加载本地资源或 HTTPS 资源。

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'none';">
```

如果出现类似错误，通常因为未设置 CSP：

```
Loading the font '<URL>' violates the following Content Security Policy directive: "default-src 'none'". Note that 'font-src' was not explicitly set, so 'default-src' is used as a fallback. The action has been blocked.
```

允许加载**本地**的 JS、CSS、图片、字体资源文件，允许加载 HTTPS、Base64 的图片，允许加载 Base64 的字体。

```html
<meta
  http-equiv="Content-Security-Policy"
  content="default-src 'none'; img-src ${webview.cspSource} https: data:; script-src ${webview.cspSource}; style-src ${webview.cspSource}; font-src ${cspSource} data:;"
/>
```

- [webview.cspSource](https://code.visualstudio.com/api/references/vscode-api#extensions) 最终会注入类似 `font-src 'self' vscode-remote-resource: vscode-managed-remote-resource: https://*.vscode-unpkg.net` 的规则，以加载本地资源。
- `https:` 顾名思义
- `data:` 表示允许使用 Base64 资源。一些构建工具在打包时，若文件大小小于特定值会以 Base64 的形式打包到 JS 或 CSS 文件中

> 完整示例参考 [webview-sample](https://github.com/microsoft/vscode-extension-samples/tree/main/webview-sample)
