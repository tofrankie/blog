## 背景

当我们开发完一个 Raycast 扩展，如要发布到 Raycast Store，需要在官方[仓库](https://github.com/raycast/extensions)提交 PR，等待官方成员审核合并。

但有一个令人头疼的事情，Raycast Store 现有扩展数 2000+，每个扩展都要有截图信息，使得整个仓库非常大（超过 70GB）。无论克隆，还是本地空间占用都极其不友好。

## 解决方案

Git 提供了不同克隆方案：

- 浅克隆（Shallow Clone）：`git clone --depth <N> <repo_url>` 只拉取最近 N 次的提交记录，而不是完整的历史。
- 部分克隆（Partial Clone）：`git clone --filter=blob:none <repo_url>` 它会下载所有的提交历史，但不会立即下载内容。只有切换分支或者查看某个文件时，它才会按需从远程仓库下载。
- 稀疏检出（Sparse Checkout）：`git clone --filter=blob:none --no-checkout <repo_url>` 适合只需要修改仓库某个子目录。

常规克隆会下载所有 Commit 对象 + 所有 Blob 对象。部分克隆会下载所有 Commit 对象，但跳过所有的 Blob 对象。其中 Commit 对象包含作者、时间、提交记录、树结构，这些信息通常是文本，增长速度很慢，一般不会太大，最多也就几百 MB。

无论是新增或者更新扩展，只需要调整 `extensions` 下某个子目录，所以这里选择「部分克隆 + 稀疏检出」结合的方案。

## 部分克隆 + 稀疏检出

> 结果：从 70GB+ → 200MB，舒服了 😜

### 简要了解

1. 只拉取记录，不拉取文件：`git clone --filter=blob:none --no-checkout <url>`
2. 按需检出子模块，工作区仅显示你指定的文件或目录：`git sparse-checkout init --cone` + `git sparse-checkout set <file_or_directory>`
3. 执行 checkout 下载指定的文件或目录：`git checkout main`
4. 后续在现有基础上最佳新的目录：`git sparse-checkout add <new_file_or_directory>`
5. 重新设定查看范围：`git sparse-checkout set <new_file_or_directory>`
6. 禁用稀疏模式，恢复所有文件： `git sparse-checkout disable`

### main 分支示例

官方 [extensions](https://github.com/raycast/extensions) 仓库目录如下，完整克隆时 `.git` 占用超过 55GB，`extensions` 占用超过 15GB。

```bash
$ tree . -L 1
.
├── .git
├── .gitattributes
├── .github
├── .gitignore
├── .nvmrc
├── CONTRIBUTING.md
├── docs
├── examples
├── examples
├── images
├── LICENSE
├── migrations
├── README.md
├── scripts
├── SECURITY.md
└── templates
```

因此，部分克隆的时候，排除掉 `extensions` 目录便可。

1. 部分克隆：不下载文件实体，不自动检出

```bash
$ git clone --filter=blob:none --no-checkout git@github.com:tofrankie/raycast-extensions.git extensions

$ cd extensions
```

2. 初始化稀疏检出

```bash
$ git sparse-checkout init --cone
```

3. 检出“除 extensions 目录外”的所有内容

```bash
$ git sparse-checkout set $(git ls-tree -d --name-only HEAD | grep -v '^extensions$')
```

> 由于此时本地还有任何文件（除 `.git` 外），不能使用 `$(ls -d */ | grep -v 'extensions/' | sed 's/\///g')` 等命令过滤，避免找不到任何文件夹抛出错误。

> 使用 `git ls-tree` 获取远程根目录下的所有文件夹，过滤掉 `extensions`

4. 执行检出，下载文件实体

```bash
$ git checkout main
```

> 此时本地仓库还没有 `extensions` 目录

5. 后续想要查看某个扩展（不影响已经存在的其他目录）

```bash
$ git sparse-checkout add extensions/wechat-devtool
```

> 不影响已经存在的其他目录，Git 会自动下载 `extensions/wechat-devtool` 的 Blob 数据。

6. 如果只是想临时查看某个文件（不会下载到工作区，直接输出在终端）

```bash
$ git show HEAD:extensions/wechat-devtool/package.json
```

### 新分支

有时候，可能会同时维护着多个扩展，加之官方目前审核速度极慢，所以个人习惯维护多个子分支，一个分支对应一个扩展的维护，而 `main` 分支负责跟 `upstream` 同步，因此它总是最新的。

新开分支之前，先从上游同步到我的 `main` 分支，接着基于 `main` 分支创建新的分支。

以 `ext/my-new-extension` 为例：

1. 创建并切换新分支

```bash
$ git checkout -b ext/my-new-extension
```

> 由于是基于 `main` 分支创建的，所以 `main` 分支可以看到的目录，新分支也能看到。

> 如果你希望切换到 `main` 分支只能看到 A，切换到 `B` 分支，只能看到 B，你可以在每次切换分支后执行一次 `git sparse-checkout set ...`。这比较麻烦，通常不建议。Raycast 扩展之间是独立的包，不会相互影响，而且几个扩展所占的空间不大（相比整体超过 70GB 来说），没必要。

2. 添加新扩展的目录到稀疏检出列表（即便这个目录在远程中不存在）

```bash
$ git sparse-checkout add extensions/my-new-extension
```

3. 后续可以正常 commit、push 等操作
