## 前言

此前，要使用 Raycast 内置的 AI 功能需要订阅 Raycast Pro。甚至需要同时开通 Raycast Pro 和大模型厂商订阅，花两份钱。在 macOS Tahoe 发布的 Spotlight 迎来史上最大更新，Raycast 倍感压力，开放了一些能力给免费订阅用户。

时间线：

- Raycast v1.102.0 带来了 Bring Your Own Models (BYOM)
- Raycast v1.100.0 带来了 Bring Your Own Key (BYOK)，允许用户使用自己的 Anthropic、Google 和 OpenAI 的 API Key，无需订阅 Raycast Pro，但不支持自定义 API URL。
- Raycast v1.99.0 带来了 Local Models，支持使用本地 Ollama 模型。

> [Raycast Changelog](https://www.raycast.com/changelog)

## 基于 Custom Provider 的中转服务 👍

### 开始之前

- 不低于 Raycast v1.102.0
- 无需订阅 Raycast Pro

### Raycast 配置

1. 打开 Raycast Settings AI 界面
2. 启用 Custom Providers 选项
3. 点击 Custom Providers 的 Reveal Providers Config，打开 `~/.config/raycast/ai` 目录
4. 拷贝 `providers.template.yaml` 一份，命名为 `providers.yaml`
5. 修改 `providers.yaml` 配置
6. 依次修改 Quick AI Model、New Chat Settings、AI Commands Model，选择你的 Custom Provider Model。
   - 个人习惯将 New Chat Settings 选用 Last Used Model
7. 后续也可以在 AI Command 的 Action Panel 中切换模型。

示例：

```yaml
# ~/.config/raycast/ai/providers.yaml
providers:
  # provider with multiple api keys
  - id: custom_provider
    name: Custom Provider
    base_url: <your-api-base-url>
    api_keys:
      openai: <your-api-key>
    models:
      - id: gpt-5.4
        name: 'GPT-5.4'
        context: 200000
        provider: openai # matches "openai" in api_keys
        abilities:
          temperature:
            supported: true
          vision:
            supported: true
          system_message:
            supported: true
          tools:
            supported: true

```

## 基于 Ollama 的中转服务

原理是代理 Ollama 的 API 调用转至任意兼容 OpenAI 接口的服务提供商。

本方案使用 [raycast-ai-openrouter-proxy](https://github.com/miikkaylisiurunen/raycast-ai-openrouter-proxy) 项目，关于如何使用文档已经写得很好了。

### 开始之前

要求：

- 不低于 Raycast v1.99.0
- 无需订阅 Raycast Pro
- 无需安装 Ollama
- 需要安装 Docker

不支持特性：

- 无法使用 Web Search
- 不支持 Remote AI Extension

### 启动 raycast-ai-openrouter-proxy

1. 克隆到本地

```bash
$ git clone https://github.com/miikkaylisiurunen/raycast-ai-openrouter-proxy.git

$ cd raycast-ai-openrouter-proxy
```

2. 修改 `docker-compose.yml` 文件，把 `YOUR_API_KEY` 和 `BASE_URL` 修改为你的 Api Key 和 baseURL。

3. 修改 `models.json` 配置文件。示例

```jsonc
[
  {
    "name": "GPT-5.4", // 在 Raycast 显示的模型名称
    "id": "openai/gpt-5.4", // 按照你的服务提供商期望格式标注的型号 ID
    "contextLength": 128000, // 模型支持的最大上下文长度（以 token 计）。它只影响 Raycast 的 UI，不影响模型本身。
    "capabilities": ["vision", "tools"] // 表示模型能力，vision 表示可以处理图像，tools 表示可以支持 AI Extensions 和 MCP
  }
]
```

> [更多细节](https://github.com/miikkaylisiurunen/raycast-ai-openrouter-proxy#configuration)

- 使用 `tools` 需要在 Raycast AI 设置最下面开启「AI Extensions for Ollama Models」选项。
- 注：Ollama 方案不支持 Remote AI Extension。
- 每次修改 models.json 需要重启代理服务 `docker compose restart`

4. 构建并启动 Docker 服务

```bash
$ docker compose up -d --build
```

其他命令：

```bash
# 启动服务
$ docker compose up -d

# 停止服务
$ docker compose down

# 重启服务
$ docker compose restart

# 查看 compose 容器状态
$ docker compose ps

# 查看 compose 近期日志
$ docker compose logs --tail=200

# 查看 compose 配置
$ docker compose config
```

5. 确保 Docker 服务已启动，日志里出现类似 `Server is up on port 3000` 说明容器内服务正常启动。

```bash
$ docker compose ps
$ docker compose logs --tail=200
```

### 配置 Raycast

> 请确保不低于 Raycast v1.99.0，且上述 Docker 服务已启动。

1. 打开 Raycast Settings AI 界面
2. 启用 AI Extensions for Ollama Models 选项
3. 设置 Ollama Host 为 `localhost:11435`（如果前面有修改 raycast-ai-openrouter-proxy 默认端口号，按需调整）
4. 点击 Sync Models，预期应该会有 `Synced Ollama Models` 的提示，但可能不会直接显示
5. 依次修改 Quick AI Model、New Chat Settings、AI Commands Model，选择你的 Local Model。
   - 个人习惯将 New Chat Settings 选用 Last Used Model
6. 后续也可以在 AI Command 的 Action Panel 中切换模型。

### 不用 Docker 的方案

其实 raycast-ai-openrouter-proxy 就是一个 Node 服务，可以不用 Docker 启动。

1. 创建 `.env` 文件： `cp .env.example .env`
2. 修改 `.env`

```
PORT="11435"
API_KEY="<your-api-key>"
BASE_URL="<your-api-base-url>"
MODELS_PATH="./models.json"
```

> 不使用 Docker 的话，这里改用本地端口 11435。

3. 安装依赖 `npm install`
4. 构建 `npm run build`
5. 启动服务 `npm run start`
