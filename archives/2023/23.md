> 以下操作，需先将虚拟机关机。

## 解决方案

1. 在 Parallels Desktop 的控制中心，进入对应系统配置页面。

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2023/10/1696414583463.png)

2. 打开「硬件 — CPU 与内存 — 高级」选项：

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1767198246491.png)

3. 将虚拟机监控程序修改为「Apple」，并保存。

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1767198255522.png)

4. 打开系统「终端 Terminal」软件，并执行以下命令：

```shell
$ sudo launchctl stop com.apple.audio.coreaudiod && sudo launchctl start com.apple.audio.coreaudiod
```

> 若系统版本低于 macOS Catalina 10.15.3，请使用 `kill coreaudiod` 命令。

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1767198290664.png)

## More

如果以上步骤完成并保存之后，发现无法打开虚拟机，提示如下：

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1767198308409.png)

> 目前问题暂时无法解决，只能将前面的「虚拟机监控程序」改回「Parallels」。

网上有些文章给出的方案：

```shell
# 1. 列出显示虚拟机信息
$ prltcl list -a
UUID                                    STATUS       IP_ADDR         NAME
{CT_ID}                                  stopped      -               Windows 11

# 2. 设置容器参数
$ prlctl set <CT_ID> --hypervisor-type parallels
The VM has been successfully configured.
```

这两个操作也只是将「虚拟机监控程序」改回「Parallels」而已。

![](https://cdn.jsdelivr.net/gh/tofrankie/blog@main/images/2026/1/1767198331660.png)
